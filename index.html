This is a massive and exciting undertaking. Building a system of this scale requires balancing hyper-optimized user experiences on the client side with aggressive, asynchronous processing on the backend.

Here is the comprehensive architectural and product blueprint for CosmoClips, a fully original, production-ready vertical video platform.

1. Product Overview

App Name: CosmoClips
Core Philosophy: "Immersive connection, zero friction." CosmoClips strips away heavy UI chrome to focus entirely on full-bleed, high-definition content, utilizing gesture-based interactions rather than traditional tapping.

User Personas:

  • The Drifter (Consumer): Wants immediate entertainment with zero cognitive load. Bounces quickly if content isn't relevant.
  • The Artisan (Creator): Values high-quality video pipelines, precise control over their uploads, and deep analytics.
  • The Connector (Engager): Uses the platform to communicate with friends, heavily utilizing the share and private-message features.

Core User Flow:

  1. Onboarding: 3-second interest selection (floating bubbles) → Immediate drop into the "Pulse" (Feed).
  2. Consumption: Vertical swipe to skip. Press and hold to pause. Double tap to "Spark" (Like).
  3. Upload: Tap the floating Action Ring → Select media → Auto-trim → Async upload via pre-signed URL while the user returns to the feed.

2. Design System & UI/UX

We must avoid copying the standard "right-aligned vertical button stack."

  • Original Interface Layout:
    • The Content: 100% edge-to-edge video. No safe-area blocking.
    • The "Halo" Nav: A dynamic, thumb-accessible semicircular dial at the bottom center of the screen. Swiping the dial left/right switches between "Following," "Pulse" (Trending), and "Local."
    • Interaction Drawer: Swiping left on the video pulls out the "Context Sheet" (glassmorphic overlay containing comments, shares, and the creator's profile ring).
  • Color Palette:
    • Background: Pitch Black (#050505) - True black for OLED power saving.
    • Primary Accent: Hyper-Violet (#7B2CBF) - Used for active states and the Action Ring.
    • Secondary: Cyan Glow (#00F5D4) - Used for notifications and "Sparks" (Likes).
  • Typography:
    • Headings: Space Grotesk (Modern, slightly geometric for creator handles).
    • UI/Body: Inter (Highly legible at small sizes for comments).

3. System Architecture

We will use a Modular Monolith transitioning to Microservices. Starting with a modular monolith in Go allows for high iteration speed, which can be cleanly broken out into microservices as scaling limits are hit.

Text-Based Architecture Diagram:

  • Storage Strategy:
    • PostgreSQL: Relational data (User profiles, settings, follow graph).
    • Cassandra / ScyllaDB: High write-throughput data (Likes, comments, feed history).
    • Redis: Caching the hot-path feed, session data, and rate-limiting counters.

4. Tech Stack

Layer Technology Justification
Frontend Flutter Compiles to native ARM code. Unparalleled control over the render pipeline via Impeller, crucial for jank-free 60fps video scrolling.
Backend Core Go (Golang) Incredible concurrency model (Goroutines). Low memory footprint, ideal for handling thousands of simultaneous feed connections.
ML / Algo Python (PyTorch) Industry standard for building the recommendation engine and two-tower embeddings.
Video Pipeline AWS Elemental + FFmpeg Hardware-accelerated transcoding to generate HLS streams rapidly.
Streaming HLS (HTTP Live Streaming) Adaptive bitrate streaming ensures smooth playback on slow cellular networks.

5. Recommendation Engine

CosmoClips uses a Two-Tower Neural Network architecture (Hybrid collaborative filtering + content-based).

The Scoring Model:
To rank videos for a user's feed session, the system calculates an expected engagement score.

$$E(v_i | u_j) = w_1 \cdot P(WatchTime) + w_2 \cdot P(Spark) + w_3 \cdot P(Share) - \lambda \cdot P(Skip)$$

  • Exploration vs Exploitation: We use an \(\epsilon\)-greedy approach. 80% of the feed is optimized for max engagement (Exploitation), 20% injects random high-quality or new content to map the user's shifting interests (Exploration).
  • Cold Start: New users receive a "Global Top 100" cluster, segmented by locale. Their first 10 swipes drastically shift their initial vector embedding.

7. Video Pipeline

  1. Request Upload: Client requests a pre-signed S3 URL from the Go backend.
  2. Direct-to-S3: Client uploads the raw .mp4 directly to an AWS S3 Raw Bucket (bypassing our API servers).
  3. Event Trigger: S3 emits an EventBridge event to an AWS SQS queue.
  4. Transcoding: A fleet of spot instances picks up the queue. FFmpeg generates HLS playlists (.m3u8) and chunks (.ts) at 1080p, 720p, and 480p, plus a WebP thumbnail.
  5. Publish: Processed assets are moved to a public S3 bucket fronted by a CDN (Cloudflare). A webhook notifies the Go backend to mark the video as "Active".

8. Scalability & Performance

  • Handling 10M+ Users:
    • Pre-computing Feeds: We do not calculate the feed on the fly. Asynchronous background workers generate personalized feed queues and store them in Redis. When the client calls /feed, it just pops from the Redis list.
    • Read Replicas: PostgreSQL uses heavy read-replicas since social media is a 95% Read / 5% Write workload.
    • Consistent Hashing: Used to route specific video comment streams to dedicated WebSocket servers to prevent hot-partition bottlenecks.

9. Security & Safety

  • Content Moderation:
    • Automated: All uploaded frames are sampled (1 frame per second) and run through an AI vision model (like AWS Rekognition) to detect NSFW content, violence, or hate symbols. Audio is transcribed and checked for banned keywords.
    • Human: Content flagged with a confidence score between 60%-90% is routed to a human review queue before broad distribution.
  • Abuse Detection: IP reputation tracking and device fingerprinting are used to shadowban bot nets attempting to manipulate the algorithm via fake "Sparks".

import (

// Video structure mapped to DB

	app.Use(logger.New())

	v1 := app.Group("/v1")
	
	// Feed Endpoint
	v1.Get("/feed/pulse", func(c *fiber.Ctx) error {

B. Database Schema (PostgreSQL)

CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    handle VARCHAR(30) UNIQUE NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE videos (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    author_id UUID REFERENCES users(id),
    status VARCHAR(20) DEFAULT 'processing', -- processing, active, flagged
    manifest_url TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Index for fast user profile video lookups
CREATE INDEX idx_videos_author ON videos(author_id, created_at DESC);

C. Frontend Skeleton (Flutter)

import 'package:flutter/material.dart';
import 'package:video_player/video_player.dart'; // Abstraction over native players

class PulseFeed extends StatefulWidget {
  @override
  _PulseFeedState createState() => _PulseFeedState();
}

class _PulseFeedState extends State {
  final PageController _pageController = PageController();
  List videoManifests = []; // Fetched from Go API

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      backgroundColor: Colors.black,
      body: PageView.builder(
        scrollDirection: Axis.vertical,
        controller: _pageController,
        itemCount: 10, // Dynamic in production
        itemBuilder: (context, index) {
          // VideoPlayerWidget handles HLS streaming and caching natively
          return Stack(
            children: [
              VideoPlayerWidget(url: videoManifests[index]),
              _buildInteractionDrawer(), // The side-swipe glassmorphic UI
            ],
          );
        },
      ),
    );
  }

  Widget _buildInteractionDrawer() {
    return Positioned(
      bottom: 50,
      right: 16,
      child: Column(
        children: [
          IconButton(
            icon: Icon(Icons.bolt, color: Color(0xFF00F5D4)), // 'Spark' Like
            onPressed: () => print("Sparked!"),
          ),
          IconButton(
            icon: Icon(Icons.comment, color: Colors.white),
            onPressed: () => print("Open Context Sheet"),
          ),
        ],
      ),
    );
  }
}
